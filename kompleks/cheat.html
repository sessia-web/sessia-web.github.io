📈 Блок 1 · Математические методы оптимизации

1.1 Что такое линейное программирование и какие основные элементы включает его постановка?
ЛП: экстремум линейной Z = Σcj·xj при Σaij·xj ≤ (=,≥) bi, xj ≥ 0. Элементы: переменные, ЦФ, ограничения, неотрицательность, данные (c,A,b). Оптимум — в вершине выпуклой ОДР. Канторович-1939, Данциг-1947.
1.2 Как составить математическую модель задачи линейного программирования?
Алгоритм модели: переменные (с единицами) → ЦФ Z = Σcj·xj → max/min → ограничения Σaij·xj ≤ (=,≥) bi → xj ≥ 0 → каноническая форма (+si). si — остаток ресурса; min Z ⇔ max(−Z).
1.3 Приведите пример задачи, которая сводится к задаче линейного программирования.
Шихтовка (задача о смесях): Z = 300x1 + 200x2 → min; x1 + x2 = 100; 0,6x1 + 0,4x2 ≥ 50 ⇒ x1 = x2 = 50, Zmin = 25 000 руб. Схема сведения: переменные → линейная ЦФ → линейные ограничения.
1.4 Что такое целевая функция и какие требования предъявляются к ней в задачах линейного программирования?
ЦФ: Z = Σcj·xj → max/min — критерий оптимальности плана. Требования: линейность (пропорциональность + аддитивность), один критерий, единые единицы, детерминированные cj. min Z ⇔ max(−Z); оптимум — в вершине ОДР.
1.5 Объясните, как работает графический метод решения задач линейного программирования для двух переменных.
ОДР = пересечение полуплоскостей; строим ∇F=(c1;c2), линию уровня c1x1+c2x2=const двигаем по градиенту до последней точки ОДР → вершина-оптимум (координаты из системы 2 уравнений). Исходы: 1 решение / сторона (∞ решений) / F не ограничена / ОДР пуста.
1.6 В чем заключается разница между графическим методом и симплекс-методом решения задачи линейного программирования?
Общее: оптимум в вершине ОДР. Графический: n=2, наглядно, линия уровня по градиенту. Симплекс: любое n, каноническая форма, опорное решение = вершина, направленный перебор смежных вершин; оптимум при Δj = zj − cj ≥ 0 (max).
1.7 Какие типы задач можно решать с помощью графического метода линейного программирования?
Графически решаются: ЗЛП с n = 2 (ресурсы → max; смеси/диета → min); задачи с n − m ≤ 2 (через 2 свободные переменные); игры 2×n и m×2 (огибающие); анализ чувствительности. Ограничение: только 2 переменные, точность чертежа.
1.8 Что такое сетевой график и какие основные элементы входят в его структуру?
Сетевой график = ориентированный граф без циклов: работы (стрелки: действит./ожидание/фиктивная t=0), события (кружки, без длительности), пути. t(L)=Σt(i,j); критический путь t(кр)=max t(L) — определяет срок проекта.
1.9 Объясните, что такое метод критического пути (CPM) и как он используется для оптимизации процессов.
CPM: прямой проход T(р)(j)=max[T(р)(i)+t(i,j)], обратный T(п)(i)=min[T(п)(j)−t(i,j)], резерв R=T(п)−T(р); критический путь — R=0, его длина = срок проекта. Оптимизация: ускорять/усиливать только критические работы, резервы некритических — на выравнивание ресурсов.
1.10 В чем заключается роль событий и работ в сетевых графиках?
Работа = процесс (действительная / ожидание / фиктивная t=0), носитель времени t(i,j). Событие = мгновенный факт завершения работ, узел графа; T(р)(j)=max[T(р)(i)+t], T(п)(i)=min[T(п)(j)−t], R(i)=T(п)−T(р), R=0 → критический путь.
1.11 Как определить критический путь в сетевом графике?
Критический путь = max по полным путям, R = 0. Прямой: T(р)(j)=max[T(р)(i)+t(i,j)]; обратный: T(п)(i)=min[T(п)(j)−t(i,j)]; R(i)=T(п)−T(р); R(п)(i,j)=T(п)(j)−T(р)(i)−t(i,j). Пример: (1,2)=5,(1,3)=3,(2,4)=6,(3,4)=2 → 1→2→4, t(кр)=11.
1.12 Какие преимущества имеет использование сетевого планирования в управлении проектами?
СПУ: явные взаимосвязи работ (в отличие от Ганта), расчётный срок t(кр)=max t(L), принцип ведущего звена (критических работ ~10–20%), резервы R(п)=T(п)(j)−T(р)(i)−t — манёвр ресурсами, оптимизация время—стоимость, пересчёт по факту, автоматизация (MS Project, Primavera).
1.13 Как построить первоначальный план для транспортной задачи методом северо-западного угла?
ТЗ: F = ΣΣ cij·xij → min, Σj xij = ai, Σi xij = bj, баланс Σai = Σbj. Опорный план = m+n−1 клеток. СЗУ: с клетки (1,1), x = min(a,b), вычеркнуть ряд, сдвиг вправо/вниз; тарифы НЕ учитывает → далее метод потенциалов.
1.14 Какие методы решения транспортной задачи существуют, кроме метода северо-западного угла?
Старт: мин. элемент, Фогель (штраф = разность 2 мин. тарифов ряда), двойное предпочтение. Оптимизация: метод потенциалов — ui + vj = cij (базис), Δij = cij − ui − vj ≥ 0 ⇒ оптимум, иначе цикл пересчёта. Ещё: распределительный, венгерский (назначения).
1.15 В чем заключается разница между открытой и закрытой транспортной задачей?
Закрытая ТЗ: Σai = Σbj — разрешима всегда; открытая: Σai ≠ Σbj. Сведение: фиктивный потребитель (избыток) или фиктивный поставщик (дефицит) на разность баланса, тарифы 0. Смысл: остатки на складе / недопоставка (вместо 0 можно штрафы).
1.16 Какие проблемы могут возникнуть при решении транспортной задачи, и как их можно решить?
Проблемы ТЗ: небаланс → фиктивный ряд (тарифы 0); вырожденность (< m+n−1 клеток) → базисный нуль / ε; запрет маршрута → тариф M; xij ≤ dij → ТЗ с огранич. пропускными способностями; Δij = 0 → альтернативные оптимумы. Целочисленность — автоматически.
1.17 Как можно преобразовать прямую задачу в двойственную?
Прямая: max Z=Σcj·xj, Ax≤b, x≥0 ⇄ Двойств.: min W=Σbi·yi, AT·y≥c, y≥0. Правила: max↔min, c↔b, A→AT, ≤→≥, ограничения↔переменные (m перем., n огранич.); «=» в прямой → yi любого знака. Двойственная к двойственной = прямая.
1.18 В чем заключается экономическое содержание теоремы двойственности?
1-я теорема: Zmax = Wmin — max прибыль = min суммарная оценка ресурсов; yi* = ΔZmax/Δbi — теневая цена (оценка Канторовича). 2-я теорема: yi*(bi−Σaij·xj*)=0, xj*(Σaij·yi*−cj)=0 — недоиспользованный ресурс → оценка 0; продукт, убыточный в оценках, → не выпускается.
1.19 Что такое производственная функция и как она моделирует зависимость между ресурсами и результатами производства?
ПФ: Q = f(x1,…,xn) — максимальный выпуск при данных ресурсах. Свойства: f(0)=0, ∂f/∂xi≥0, ∂²f/∂xi²<0. MPi=∂f/∂xi, APi=Q/xi, Ei=MPi·xi/Q. Кобб–Дуглас: Q = A·Kα·Lβ.
1.20 Какие основные типы производственных функций существуют?
Типы ПФ по σ: линейная Q=Σaixi (σ=∞); Леонтьев Q=min(xi/ai) (σ=0); Кобб–Дуглас Q=A·Kα·Lβ (σ=1, α+β — масштаб); CES Q=A(αK−ρ+(1−α)L−ρ)(−1/ρ), σ=1/(1+ρ); с НТП: множитель e(γt).
1.21 Как анализировать эффективность использования ресурсов с помощью производственных функций?
Эффективность через ПФ: AP=Q/x (фондоотдача, производительность), MP=∂Q/∂x, E=MP·x/Q (у Кобба–Дугласа = α, β). Оптимум: MP1/w1 = MP2/w2; наращивать ресурс пока p·MP ≥ w. Масштаб: α+β; техпрогресс — рост A.
1.22 Какие факторы могут влиять на форму производственной функции в экономике?
Факторы формы ПФ: технология/взаимозаменяемость (σ: 0 — Леонтьев, 1 — Кобб–Дуглас, ∞ — линейная), НТП (A·e(γt), сдвиг α, β), масштаб (α+β), качество ресурсов, природно-отраслевые условия, горизонт (кратко-/долгосрочный), агрегирование.
1.23 Как использовать производственные функции для оптимизации производственных процессов?
Оптимизация по ПФ: max f(x) при Σwixi ≤ C → метод Лагранжа → MP1/w1 = MP2/w2 = λ (λ — отдача рубля бюджета); min издержек при f(x)=Q0 → MRTS = w1/w2; max прибыли → p·MPi = wi. Геометрия: касание изокванты и изокосты.
1.24 Каковы основные этапы анализа сетевого графика для оптимизации времени выполнения проекта?
Анализ сети: построение → прямой ход tр(j)=max{tр(i)+t(i,j)} → обратный tп(i)=min{tп(j)−t(i,j)} → резервы R=tп−tр → критический путь (R=0) → оптимизация (ресурсы с резервных работ на критические).
1.25 Что такое резерв времени в сетевом графике и как его вычислить?
Резерв — допустимая задержка без срыва Tкр. R(i)=tп(i)−tр(i); Rп(i,j)=tп(j)−tр(i)−t(i,j); Rс(i,j)=tр(j)−tр(i)−t(i,j); Rс≤Rп; на критическом пути все резервы = 0.
1.26 Объясните, что такое пути в сетевом графике и как они используются для оптимизации процессов.
Путь — цепочка работ «конец = начало следующей»; t(L)=Σt(i,j); критический — max длина = Tкр; R(L)=Tкр−t(L); Kн=(t(Lmax)−t′кр)/(Tкр−t′кр). Оптимизация: ресурсы с резервных путей → на критический.
1.27 Как определить критический путь в сетевом графике и какую роль он играет в проектном управлении?
Критический путь — max по длине полный путь; Tкр = min срок проекта. Поиск: tр(j)=max{tр(i)+t}, tп(i)=min{tп(j)−t}, критические работы: Rп=0. Задержка критической работы = задержка всего проекта.
1.28 Какие методы используются для оптимизации сетевых графиков?
Оптимизация: по времени (ресурсы с резервных работ → на критические, запараллеливание), по ресурсам (сдвиг в пределах резервов = сглаживание пиков), по стоимости (сжимать критические работы с min k = (Cу−Cн)/(tн−tу)); PERT: tож=(tmin+4tнв+tmax)/6.
1.29 Что такое теория игр и как она применяется в экономике?
Теория игр — мат. теория конфликтных ситуаций (раздел исследования операций). α = maxi minj aij ≤ v ≤ β = minj maxi aij; α = β → седловая точка, чистые стратегии. Экономика: олигополия (Курно/Бертран), картели (дилемма заключённого), аукционы, игры с природой.
1.30 Как определить стратегию для игроков в теории игр?
Стратегия — правило выбора хода в каждой ситуации. Схема: матрица → доминирование → седло (α = β)? → иначе смешанные. 2×2: p1 = (a22 − a21)/(a11 + a22 − a12 − a21); v = (a11·a22 − a12·a21)/(a11 + a22 − a12 − a21).
1.31 Какие методы анализа игр применяются для нахождения оптимальных решений?
Каскад: доминирование → седло (α = β) → 2×2-формулы / графика (2×n: max нижней огибающей; m×2: min верхней) → ЛП: xi = pi/v, min Σxi при Σaij·xi ≥ 1, v = 1/Σxi → Браун—Робинсон. Природа: Вальд max min; Сэвидж min max риска (rij = max aij − aij); Гурвиц λ; Байес max Σ pj·aij.
1.32 Что такое матричная игра и как ее можно преобразовать в задачу линейного программирования?
Матричная игра = антагонист. игра 2 лиц, матрица A. α = max min ≤ β = min max; нет седла → смешанные p, q. Замена xi=pi/v: Z=Σxi→min, Σaijxi≥1, x≥0; v=1/Z*, pi=v·xi (двойственная — для 2-го игрока).
1.33 Объясните, как составить математическую модель для матричной игры.
Модель: игроки → стратегии Ai, Bj → матрица aij → доминирование → α = max min, β = min max → седло или смешанные p, q. 2×2: p1*=(a22−a21)/D, v=(a11a22−a12a21)/D, D=a11+a22−a12−a21.
1.34 Как теория игр может быть использована для решения задач линейного программирования?
Игра ↔ пара двойственных ЛП. ЛП → симметричная игра: P = [[0, A, −b], [−AT, 0, c], [bT, −cT, 0]], v = 0; t > 0 → x̄ = x*/t (Данциг). Брауна—Робинсон = приближённое решение ЛП; теневые цены = стратегия «рынка»; максимин = критерий Вальда.
1.35 Приведите пример задачи теории игр, которая сводится к линейному программированию.
Пример: две фирмы, A = [[2,3,4],[4,2,3],[3,4,2]], α=2 < β=4 → нет седла. ЛП: Σxi→min, Σaijxi≥1 → xi=1/9, Z*=1/3, v=3, p=q=(1/3,1/3,1/3); проверка: (2+4+3)/3=3.
1.36 В чем заключается связь между стратегиями игроков в теории игр и методами решения задач линейного программирования?
Игра A(m×n), aij>0: замена xi = pi/v → min Σxi при Σaij·xi ≥ 1; задача игрока 2 — двойственная; v = 1/Σxi*, pi* = v·xi*, qj* = v·yj*. Минимакс фон Неймана ⇔ теорема двойственности ЛП.
1.37 Какие задачи теории игр могут быть решены с помощью методов линейного программирования?
ЛП решает все матричные игры (2 игрока, нулевая сумма, конечные): m×n → пара двойственных ЗЛП, v = 1/Σxi*. Седловая точка — чистые стратегии; 2×2 — формулы; 2×n/m×2 — графика; биматричные и кооперативные — НЕ ЛП.
1.38 Как игровые модели используются для анализа экономических стратегий?
Игра = (игроки, стратегии Si, выигрыши ui). Анализ: доминирование → максимин → равновесие Нэша (одностороннее отклонение невыгодно) → Парето. Ценовая война = дилемма заключённого: равновесие (3;3) хуже кооперации (5;5).
1.39 В чем заключается роль теории игр в стратегическом планировании и экономике?
Роль ТИ: максимин (гарантия α = maxi minj aij), равновесие Нэша (прогноз конкурентов), игры с природой — Вальд max min, Сэвидж min max (βj − aij), Гурвиц λ·max+(1−λ)·min, Лаплас (среднее); коалиции — вектор Шепли.
1.40 Как можно применить игровую теорию для анализа конкурентных рынков?
Олигополия: Курно — объёмы, отклик q1 = (a−c−b·q2)/(2b), равновесие q* = (a−c)/(3b); Бертран — цены → p = c; Штакельберг — лидер первым. Картель = дилемма заключённого, держится только в повторяющейся игре.
1.41 Какие экономические проблемы можно моделировать с использованием теории игр?
ТИ моделирует: олигополию (Курно/Бертран), ценовые войны (дилемма заключённого), аукционы (Викри: правда — доминирующая стратегия), делёж (вектор Шепли), «принципал—агент», трагедию общин (равновесие → переэксплуатация), игры с природой (Вальд/Сэвидж/Гурвиц/Лаплас).
1.42 Как используется графический метод для решения матричных игр?
2×n/m×2 без седла (α<β): прямые Ej(p)=a1j·p+a2j·(1−p); max нижней огибающей → p*, v; активные прямые через оптимум → подыгра 2×2 → q*.
1.43 Объясните, как построить график для нахождения оптимальных стратегий в игре с двумя игроками.
Оси: p ∈ [0;1]; прямая j идёт от a2j (при p=0) к a1j (при p=1). Пересечение активных прямых: a1k·p+a2k(1−p) = a1l·p+a2l(1−p) → p*, v. Пример [(2,6),(5,3)]: p*=1/3, v=4, q*=1/2.
1.44 Какие преимущества и ограничения имеет графический метод решения матричных игр?
Плюсы: наглядно, быстро, сразу видны активные стратегии и смысл максимина. Минусы: только 2×n/m×2 (после доминирования), приближённость, вырождение. Общая игра m×n → пара двойственных ЗЛП (симплекс).
1.45 Когда графический метод применяется для решения игр с двумя переменными?
Условия: антагонистическая игра с матрицей A, нет седла (α < β), формат 2×2/2×n/m×2 — в т.ч. после вычёркивания доминируемых. Стратегия (p; 1−p) — одна переменная → решение на плоскости.
1.46 Как интерпретировать результаты, полученные с помощью графического метода, в контексте теории игр?
v — гарантированный средний выигрыш (α ≤ v ≤ β), v > 0 — выгодно A; p*, q* — частоты применения стратегий или доли ресурса; активные стратегии (прямые через оптимум) дают ровно v; равновесие: одиночное отклонение невыгодно.
1.47 Что такое классическая схема принятия решений в условиях неопределенности?
«Игра с природой»: альтернативы Ai, состояния Sj (вероятности неизвестны), платёжная матрица W = ||wij||. Схема: цель → альтернативы → состояния → матрица → критерий (Вальд/Сэвидж/Гурвиц/Лаплас) → выбор. Матрица рисков: rij = maxk wkj − wij.
1.48 Какие основные методы используются для принятия решений в условиях неопределенности?
Вальд: max min wij; Сэвидж: min max rij, rij = maxk wkj − wij; Гурвиц: max[α·min + (1−α)·max]; Лаплас: max (1/n)Σwij. Дополнительно: дерево решений, эксперты, сценарии, Монте-Карло.
1.49 В чем заключается разница между риском и неопределенностью в контексте принятия решений?
Найт: риск = вероятности известны (Bi = Σpj·wij, риск-мера — дисперсия/СКО); неопределённость = вероятности неизвестны (Вальд/Сэвидж/Гурвиц/Лаплас). Шкала: определённость → риск → неопределённость; информация сдвигает задачу влево.
1.50 Как применять критерии оптимальности при принятии решений в условиях неопределенности?
Алгоритм: матрица → критерий → свёртка строк → выбор → сверка по нескольким критериям. Вальд max min; Сэвидж min max rij (rij = maxk wkj − wij); Гурвиц α·min + (1−α)·max; Лаплас (1/n)Σ. Катастрофа → Вальд; упущенная выгода → Сэвидж.
1.51 Какую роль играют теории вероятности и статистики в принятии решений в условиях неопределенности?
Байес: Bi = Σpj·wij → max; уточнение: P(S|x) = P(x|S)·P(S)/Σ P(x|Sk)·P(Sk); EVPI = E(полная инфо) − max Bi. Статистика оценивает pj (частоты, ЗБЧ), прогнозирует состояния, Монте-Карло даёт распределение результата.

💻 Блок 2 · ИТ в управлении техническими системами

2.1 Что вы понимается под технологией программирования?
Технология программирования = совокупность производственных процессов создания ПС + их описание; каждый процесс = методы + средства + порядок. Правило Боэма: цена ошибки ×10 на каждом следующем этапе (C ≈ C0·10k).
2.2 Что такое жизненный цикл программы?
ЖЦ ПС = период от замысла до изъятия из эксплуатации. Этапы: требования (внешнее описание) → проектирование → кодирование → аттестация → эксплуатация/сопровождение (50–70% затрат). Модели: каскадная (последовательно, документы) и спиральная (Боэм: риски + прототипы по виткам).
2.3 Что такое программное средство, и какие функции оно выполняет?
ПС = программа(ы) на носителях данных + программная документация (ГОСТ 19781: программы + процедуры + правила + данные + документация). Функции: обработка данных, управление, хранение/выдача информации, интерфейс, инструментальные. Ошибка: ПС не выполняет разумных ожиданий пользователя (по документации).
2.4 Что представляет собой надежность ПС?
Надёжность ПС = способность безотказно выполнять функции при заданных условиях за заданное время с достаточной вероятностью; отказ = проявление ошибки разработки (износа нет). P(t) = e(−λt), T = 1/λ; Джелински–Моранды: λi = C·(N−i+1). Пути: предупреждение, тестирование, устойчивость к ошибкам.
2.5 Специфика разработки программных средств.
ПС = программы + документация. Специфика: неформальные требования, незримость, дуализм текст/процесс, дискретность (нет запаса прочности), сложность ~ n(n−1)/2 связей, нет износа (отказ = ошибка), ошибки неизбежны (Дейкстра); цена ошибки ×10 на этап (Боэм). Пример: Ariane 5.
2.6 Каскадная модель жизненного цикла ПС.
Каскад (Ройс, 1970): внешнее описание → конструирование → кодирование → тестирование/аттестация → сопровождение; строго последовательно, результат этапа = документ. Плюсы: план, документация, контроль. Минусы: требования заморожены, результат в конце, ошибка требований ×10…×100 (Боэм).
2.7 Усовершенствование каскадной модели ЖЦ ПС.
Усовершенствования каскада: обратные связи (Ройс) → промежуточный контроль после каждого этапа (возврат на любой этап) → V-модель (этап разработки ↔ свой уровень тестирования) → прототип при внешнем описании → инкременты/очереди (ГОСТ 34). Цель — раннее обнаружение ошибок (Боэм: цена ×10 на этап).
2.8 Спиральная модель ЖЦ.
Спираль (Боэм, 1988): виток = 4 сектора — цели/альтернативы → анализ рисков + прототип → разработка и верификация версии → планирование следующего витка; радиус ~ затраты. Управляется рисками (каскад — документами); минусы: момент завершения (тайм-боксинг), квалификация, дорого для малых проектов.
2.9 Метрология и качество ПО.
Метрология ПО — измерение характеристик ПС; качество ПС — совокупность свойств для удовлетворения потребностей пользователя. Иерархия: критерии → примитивы → метрики. V(G)=E−N+2P (Мак-Кейб), V=N·log2 n (Холстед), E=a·KLOCb (COCOMO); ГОСТ 28195, ISO 9126/25010.
2.10 Критерии качества ПО: сложность, корректность, надежность, трудоемкость.
Сложность: V(G)=E−N+2P (≤10), Холстед V=N·log2 n. Корректность = соответствие спецификации; дефекты/KLOC. Надёжность = безотказность за время t: P(t), MTBF, λ; ПО не изнашивается. Трудоёмкость: чел.-мес., COCOMO E=a·KLOCb (b>1), закон Брукса. Сложность↑ → трудоёмкость↑, надёжность↓.
2.11 Измерения и оценка качества ПО.
Измерение = получить значения метрик; оценка = сравнить с базовыми + свёртка K = Σ wi·ki (Σwi=1) ≥ порога. ГОСТ 28195: факторы→критерии→метрики→оценочные элементы. 4 метода: измерительный, регистрационный, расчётный, экспертный. Оценка: прогнозная / контрольная / фактическая.
2.12 Процесс производства ПО: методы, технология и инструментальные средства.
Производство ПО = методы (как делать: структурное/модульное программирование, нисходящая разработка) + технология (упорядоченный процесс: модели ЖЦ, ISO 12207, ГОСТ 19/34) + инструменты (IDE, CASE, Git, CI). Технология по Жоголеву = совокупность производственных процессов + её описание.
2.13 Проектирование программного обеспечения.
Проектирование ПО = ответ «как»: архитектура (цельная / комплекс автономных / слоистая / параллельная — по Жоголеву) → модульная структура (прочность max, сцепление min) → спецификации модулей. Методы: нисходящая разработка с заглушками, пошаговая детализация.
2.14 История развития ПО.
Этапы: машинные коды → ЯВУ (Fortran-1957) → кризис ПО и software engineering (НАТО-1968) → каскад (Ройс-1970) + unbundling IBM-1969 (ПО = товар) → ПК и коробочные продукты (1980-е), спираль (Боэм-1988) → Интернет → Agile-2001, SaaS. Доля ПО в стоимости систем: ~20% → 80–90%.
2.15 Типы ПО.
ПО = программы + документация. По назначению: системное / инструментальное / прикладное (общее + специальное). По режиму: пакет / диалог / реальное время. По распространению: уникальное (заказное) vs тиражируемый продукт (коммерческое / свободное / shareware / SaaS).
2.16 Уникальное ПО и ПО, как продукция.
Уникальное ПО — 1 экз. под конкретного заказчика, прямой контакт с автором; продукт — тираж анонимному пользователю, отчуждён от разработчика (полная документация, поддержка, версии). Брукс: продукт ×3, системный продукт ×9. Себестоимость копии ≈ Cразр/N + Cкопии.
2.17 Требования к ПО как к продукции. Доведение ПО до товарного уровня.
Продукт = 6 критериев качества (функциональность, надёжность, лёгкость применения, эффективность, сопровождаемость, мобильность) + документация + дистрибутив + сопровождение + лицензия/сертификация. Доведение до товара: обобщение → тестирование (альфа/бета) → документация → инсталлятор → поддержка. Брукс ×3 (×9); N* = Cдовед/(P − c).
2.18 Понятие качества ПО.
Качество ПС = совокупность черт и характеристик, влияющих на способность удовлетворять потребности пользователей. 6 критериев: функциональность, надёжность, лёгкость применения, эффективность, сопровождаемость, мобильность; критерий = набор примитивов. Уровни — в спецификации качества (часть внешнего описания); качество = достаточность, не максимум; ошибка дорожает ×10 за этап ЖЦ.
2.19 Критерии качества ПО: функциональность, надежность, их примитивы.
Функциональность = выполняет набор функций внешнего описания; надёжность = безотказно, при заданных условиях, за время t, с вероятностью P(t). Примитивы: завершённость (общая), точность, автономность, устойчивость, защищённость. Кг = То/(То+Тв); отказ ПС = ошибка разработки, не износ.
2.20 Критерии качества: легкость применения, эффективность, их примитивы.
Лёгкость применения = минимум усилий пользователя; примитивы: П-документированность, информативность, коммуникабельность (+устойчивость, защищённость). Эффективность = услуги/ресурсы; примитивы: временная, по памяти, по устройствам. Критерии конфликтуют → компромисс в спецификации качества; оптимизировать только узкие места (10% кода = 90% времени).
2.21 Критерии качества: сопровождаемость, мобильность, их примитивы.
Сопровождаемость = изучаемость (С-документированность, информативность, понятность, структурированность, удобочитаемость) + модифицируемость (расширяемость, модульность). Мобильность = перенос в другую среду; примитивы: независимость от устройств, автономность, структурированность, модульность (приём — машинно-зависимое ядро). Сопровождение ≈ 60-70% затрат ЖЦ ПС.
2.22 Преодоление барьера между пользователем и разработчиком.
Барьер = разные понятийные модели пользователя и разработчика → ошибки требований (самые дорогие: цена ×10 за этап, Боэм). Преодоление: управляемая/контролируемая пользователем разработка, прототипы, пользовательский контроль ВО, спиральная модель.
2.23 Назначение внешнего описания программного средства и его роль в обеспечении качества программного средства.
ВО = спецификация качества + функциональная спецификация (среда, функции, нештатные ситуации); точная постановка задачи и исходный документ для трёх параллельных процессов (программы, документация, тесты). Ошибки ВО — самые дорогие (×10 за этап).
2.24 Определение требований к программному средству.
Определение требований = исходный документ (задачи, цели, условия применения). Формы: словесное / с формализацией / прототип. Способы (Жоголев): управляемая пользователем, контролируемая пользователем (лучший), независимая от пользователя. Аналог: ТЗ ГОСТ 34.602, SRS.
2.25 Спецификация качества программного средства.
Спецификация качества = выбранные примитивы качества + приоритеты + проверяемая мера. 6 критериев: функциональность, надёжность (завершённость, точность, автономность, устойчивость, защищённость), лёгкость применения, эффективность, сопровождаемость, мобильность. Аналог — ISO 25010.
2.26 Методы внешнего контроля внешнего описания программного средства.
4 метода контроля ВО (Жоголев): статический просмотр; смежный контроль (сверху — от разработчиков определения требований и спецификации качества, снизу — от разработчиков архитектуры/программ/документации/тестов); пользовательский контроль; ручная имитация по тестам (вариант — за терминалом; единственный динамический).
2.27 Понятие архитектуры и задачи ее описания.
Архитектура ПС = строение извне: подсистемы + способы их взаимодействия. Задачи: выделить подсистемы, распределить функции внешнего описания, задать интерфейсы. 4 класса: цельная / комплекс автономных / слоистая (THE Дейкстры) / параллельные программы. Контроль: смежный сверху-снизу + ручная имитация.
2.28 Цель модульного программирования.
Модуль = самостоятельно оформленный, отдельно компилируемый фрагмент программы со спецификацией. Цели модульности: борьба со сложностью (7±2; связи n(n−1)/2), распределение работ, раздельная компиляция/тестирование, локализация изменений, повторное использование.
2.29 Основные характеристики программного модуля.
Характеристики модуля (Майерс): размер (V(G)=E−N+2 ≤ 10), прочность (по совпадению < логическая < функциональная < информационная), сцепление (по содержимому > по общей области > параметрическое), рутинность. Идеал: функц./информ. прочность + параметрическое сцепление + рутинный.
2.30 Методы разработки структуры программы.
Структура программы = дерево модулей (вершины — модули + спецификации, дуги — обращения). Методы: восходящая (драйверы, переделки), нисходящая (заглушки, естественное окружение), архитектурный (библиотеки), конструктивный, целенаправленная конструктивная реализация (рекомендуемый — ранний сквозной вариант).
2.31 Порядок разработки программного модуля.
Порядок разработки модуля: 1) изучение/проверка спецификации + выбор языка → 2) алгоритм и структуры данных → 3) кодирование (структурно, пошаговая детализация) → 4) шлифовка текста → 5) проверка (просмотр + сквозное прослеживание) → 6) компиляция → далее тестирование. Каждый шаг завершается контролем.
2.32 Структурное программирование.
Структурное программирование (Дейкстра, 1968): программа = композиция «следование / ветвление / цикл», один вход — один выход, без goto; основа — теорема Бёма-Джакопини (1966). Разработка — пошаговой детализацией (Вирт). Метрика: V(G) = E − N + 2 ≤ 10.
2.33 Факторы, определяющие качество ПО.
Качество ПС = совокупность черт и характеристик, обеспечивающих удовлетворение потребностей пользователей. 6 критериев: функциональность, надёжность, лёгкость применения, эффективность, сопровождаемость, мобильность; иерархия: критерий→примитив→метрика (ГОСТ 28195: фактор→критерий→метрика→оценочный элемент); P(t)=e(−λt), T=1/λ; ошибка дорожает ×10 за этап ЖЦ.
2.34 Основные понятия отладки и тестирования.
Тестирование — выполнение программы на наборе данных с заранее известным результатом (тест); отладка = тестирование + поиск + исправление. Чёрный/белый ящик; критерии покрытия: операторы, ветви (C1), пути; Дейкстра: тесты показывают наличие ошибок, не их отсутствие.
2.35 Преимущество тестирования сверху вниз.
Нисходящее тестирование: от головного модуля, подчинённые — заглушки. Плюсы: модуль в естественном окружении (драйверы не нужны), раннее обнаружение общесистемных/интерфейсных ошибок (дешевле в ~10 раз за этап: C≈C0·10k), ранний работающий каркас, многократная проверка головных модулей.

📊 Блок 3 · Статистика и практика

3.1 Статистические методы системного анализа: основные понятия, принципы и области применения
Статметоды СА = выводы о системе по выборке в условиях случайности. Совокупность/выборка, параметр(μ,σ²)/статистика(x̄,s²). x̄=(1/n)Σxi; s²=Σ(xi−x̄)²/(n−1); ДИ: x̄±t·s/√n. Применение: качество, надёжность, регрессия, прогноз.
3.2 Описательные статистики и их роль в системном анализе
Описательные статистики: положение (x̄, Me, Mo, квартили), разброс (s²=Σ(xi−x̄)²/(n−1), s, R, IQR, V=s/x̄·100%), форма (A=m3/s³, E=m4/s⁴−3). Роль в СА: сжатие данных, диагностика, выбросы (x̄±3s), выбор методов.
3.3 Оценка среднего уровня, разброса и формы распределения
Центр: x̄=(1/n)Σxi, Me, Mo; разброс: s²=Σ(xi−x̄)²/(n−1), s, V=s/x̄·100% (<33% — однородно); форма: A=m3/s³, E=m4/s⁴−3 (у нормального 0). Правило 3σ: μ±3σ → 99,73 %.
3.4 Проверка статистических гипотез о параметрах и структуре систем.
Гипотеза: H0 (нет различий) vs H1; ошибка I рода = α, II рода = β. О среднем: t = (x̄−μ0)/(s/√n) vs t(α,n−1); о дисперсиях: F = s1²/s2²; о структуре: χ² = Σ(ni−npi)²/(npi). p-value < α → отвергаем H0.
3.5 Регрессионный анализ в системном анализе: линейные, нелинейные и робастные методы оценки взаимосвязей
Регрессия: y = b0 + b1x + ε; МНК Σe² → min, b1 = Σ(x−x̄)(y−ȳ)/Σ(x−x̄)², b = (XᵀX)⁻¹Xᵀy; качество: R², F, t. Нелинейные → линеаризация/итерации; выбросы → робастные: Σ|e| → min, М-оценки Хубера.
3.6 Анализ временных рядов: тенденции, сезонность, прогностические модели
Ряд: y = T + C + S + ε (аддит.) или T·C·S·ε (мультипл.); тренд МНК ŷt = a + bt; индекс сезонности Is = ȳсез/ȳобщ; сглаживание St = αyt + (1−α)St−1; ARIMA(p,d,q), идентификация по АКФ; точность — MAPE.
3.7 Дисперсионный анализ в системном анализе
ANOVA: SSобщ = SSфакт + SSост; F = MSфакт/MSост = [SSф/(k−1)] / [SSост/(n−k)]; F > Fкр(α; k−1; n−k) → фактор значим; сила влияния η² = SSф/SSобщ. Предпосылки: нормальность + равные дисперсии; иначе Краскел–Уоллис.
3.8 Многомерная статистика в системном анализе
Многомерка: ковариационная матрица S; ГК: Sv = λv, вклад λi/Σλ, оставляем ~80% дисперсии (Кайзер: λ>1); кластеры — без учителя (k-средних, d = √Σ(ai−bi)², стандартизация!); дискриминант — с учителем; факторный: x = Λf + ε.
3.9 Байесовские методы в системном анализе: апостериорные оценки
p(θ|x) ∝ p(x|θ)·p(θ); оценки: E[θ|x] (квадр. потери), медиана, мода (MAP). Beta(a,b) + k из n → Beta(a+k, b+n−k), θ̂ = (a+k)/(a+b+n). Кредитный интервал: P(θ∈[a,b]|x)=0,95.
3.10 Байесовские методы в системном анализе: априорная информация
Априорное p(θ) — знания до опыта: информативное / неинформативное (равномерное, Джеффриса) / сопряжённое (Beta↔биномиальное, нормальное↔нормальное, гамма↔Пуассон). Нормальная модель: μpost = (τ₀μ₀ + nτx̄)/(τ₀ + nτ). Beta(a,b) ≈ a+b виртуальных испытаний.
3.11 Байесовские методы в системном анализе: адаптивное обновление знаний
«Апостериорное вчера = априорное сегодня»: pt(θ) ∝ p(xt|θ)·pt−1(θ). Beta: a += k, b += n−k. Динамика — цикл «прогноз→коррекция»; Калман: x̂ = x̂⁻ + K(z − Hx̂⁻). Нестационарность → фактор забывания.
3.12 Непараметрические методы статистики: медианные тесты
Непараметрика = без предположений о законе распределения. Критерий знаков: K ~ Bin(n; 0,5), z = (K − n/2)/(0,5√n) против 1,96. Критерий Муда: таблица 2×k «выше/ниже общей медианы» → χ² = Σ(O−E)²/E, df = k−1. АОЭ знаков = 2/π ≈ 0,64.
3.13 Непараметрические методы статистики: ранговые критерии
Ранги вместо значений (связки → средние ранги). U = n₁n₂ + n₁(n₁+1)/2 − R₁; H = 12/(N(N+1))·ΣRⱼ²/nⱼ − 3(N+1) ~ χ²(k−1); ρs = 1 − 6Σd²/(n(n²−1)). АОЭ Манна–Уитни = 3/π ≈ 0,955.
3.14 Непараметрические методы статистики: оценка устойчивости систем
Устойчивость = однородность/стационарность показателей системы. Критерий серий: M[R] = 2n₁n₂/(n₁+n₂)+1; Манн–Кендалл: S = Σsign(xⱼ−xᵢ) — тренд; КС: D = max|F̂₁−F̂₂| — любое изменение закона; робастные Me и IQR + бутстреп.
3.15 Методы статистической классификации и кластеризации: анализ группировок, деревья принятия решений, метод k-средних
Классификация — с учителем, кластеризация — без. k-средних: J = ΣₖΣ||x − μₖ||² → min, k — методом «локтя». Дерево: H = −Σp·log₂p, Джини G = 1 − Σp². Иерархия (Уорд) → дендрограмма; расстояние d = √Σ(xᵢ−yᵢ)².
3.16 Марковские процессы и анализ надёжности систем: вероятностно-статистические модели переходов состояний
Марков = без последействия: P(будущее|настоящее). π=πP; dpi/dt = вход − выход (Колмогоров). Надёжность: Kг = μ/(λ+μ) = Tо/(Tо+Tв); P(t)=e(−λt).
3.17 Метод Монте-Карло.
Монте-Карло = статистические испытания: a = M[X] ≈ (1/N)Σxi (ЗБЧ); ошибка ε ≈ 3σ/√N, N~1/ε². Розыгрыш: x = F⁻¹(r), r∈[0,1). Интеграл: I ≈ ((b−a)/N)Σf(xi).
3.18 Имитационное моделирование
Имитация (Шеннон) = модель системы + эксперименты на ней вместо реального объекта; время «проигрывается». Парадигмы: дискретно-событийная / системная динамика / агентная. Верификация≠валидация. N прогонов → x̄ ± t·S/√N.
3.19 Статистическая обработка данных в условиях неопределённости: сценарный анализ
Сценарии: песс./баз./опт., согласованные комбинации факторов. E(x)=Σpi·xi; σ²=Σpi(xi−E)²; V=σ/E. Без p: Вальд (максимин), Лаплас, Гурвиц (α), Сэвидж (минимакс сожалений).
3.20 Интеграция статистических методов системного анализа в комплексные информационно-аналитические платформы: визуализация, отчётность и поддержка принятия решений
ИАП: данные (ETL→DWH) → аналитическое ядро (OLAP, регрессия, карты Шухарта ±3σ, Марков, Монте-Карло) → представление (дашборды, отчёты, алерты). Аналитика: дескриптивная→диагностическая→предиктивная→прескриптивная. Решение — за ЛПР.
