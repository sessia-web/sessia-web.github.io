⚡ Шпаргалка перед экзаменом

Математические методы оптимизации · всё самое главное на одной странице · оценки: ≥90% — 5 · ≥75% — 4 · ≥60% — 3

⚠️ Конвенция оценок (главная ловушка)

  • Оценка свободной клетки: ΔCij = cij − (ui + vj)
  • Задача минимизации оптимальна, когда все оценки неотрицательны (ΔCij ≥ 0)
  • Отрицательная оценка → план улучшаем переносом по циклу (берём клетку с наименьшей ΔC)
  • Симплекс, задача на максимум: положительная оценка → план можно улучшить; в базис вводят столбец с наибольшей положительной оценкой
  • У базисных переменных оценки всегда 0
  • Встречается и запись w = u+v−c ≤ 0 — это то же правило, знак зависит от определения

🚚 Транспортная задача

  • Целевая функция: Z = Σ cij·xij → min
  • Переменных: m × n (3 поставщика × 4 потребителя = 12)
  • Закрытая ⇔ Σзапасов = Σпотребностей (балансовое условие — необходимо и достаточно для разрешимости)
  • Открытая → фиктивный поставщик/потребитель, тарифы нулевые, объём = разность сумм (запасы 5-10-15 < потребности 10-15-20 → открытая, фиктивный поставщик на 15)
  • Невырожденный план: занятых клеток m + n − 1; меньше → вырожденный
  • 3 этапа: ① опорный план → ② проверка оптимальности (потенциалы) → ③ улучшение по циклу
  • СЗ-угол: с левой верхней клетки, min(запас, потребность), тарифы НЕ учитывает
  • Наименьшая стоимость: клетка с min тарифом → план ближе к оптимальному
  • Потенциалы: для занятых клеток ui + vj = cij; один потенциал задаём произвольно (0)
  • Цикл пересчёта: замкнутая ломаная, вершины в занятых клетках + одна в свободной, число вершин чётное

🎒 Коммивояжёр и рюкзак (только общий вид)

  • TSP: кратчайший замкнутый маршрут через все города по одному разу
  • Маршрутов: (n−1)!/2 (симметричная задача)
  • Класс: NP-трудная; точное решение — полный перебор, метод ветвей и границ
  • Рюкзак: max Σcixi при Σwixi ≤ W; переменные xi ∈ {0, 1} («брать/не брать») → ЦЧП, NP

🧮 Классы сложности

  • P — решается за полиномиальное время; NP — решение проверяется за полином
  • NP-полная = NP-трудная и сама принадлежит NP; NP-трудная может быть и вне NP
  • Рост сложности: O(1) < log n < n < n² < 2ⁿ < n! (факториальная — максимальная)
  • O — верхняя граница (худший случай), Ω — нижняя, Θ — точная
  • К классу P: ЛП, транспортная, кратчайший путь, выпуклая оптимизация
  • NP (через ЦЧП): коммивояжёр, рюкзак, раскрой, назначения
  • Симплекс-метод в худшем случае — экспоненциальный (полиномиальны: эллипсоиды, внутренняя точка)
  • Неразрешимые задачи существуют (Тьюринг) — алгоритма нет вообще

📚 Типовые задачи по классам (учить наизусть)

  • ЛП: транспортная (классический пример!), о смесях/диете, распределение ресурсов, сетевое планирование
  • ЦЧП: раскрой, коммивояжёр, назначения, рюкзак
  • НЛП: наименьшие квадраты, оптимальное управление, квадратичные функции
  • ДП (динамическое): замена оборудования, кратчайший путь, управление запасами
  • СП (стохастическое): случайные параметры с известным распределением

🤔 Теория принятия решений

  • ТПР — дисциплина о методах обоснованного выбора наилучшего варианта
  • Формальная запись: F, A, X, G, P — Формулировка, Альтернативы, признаки (X), оГраничения, Предпочтения ЛПР
  • На стадии постановки определяют совокупность альтернатив
  • Критерий — признак, по которому производится оценка/классификация
  • Требования к критериям: полнота, разложимость, отсутствие избыточности, прозрачность
  • 5 этапов: проблемная ситуация → необходимость решения → постановка задачи → поиск альтернатив → выбор метода
  • Условия: определённость (исход известен) / риск (вероятностный) / неопределённость
  • Несколько целевых функций = многокритериальная (векторная) оптимизация
  • Проблемная ситуация — несоответствие между существующим и требуемым состоянием

📉 Градиент и спуски

  • Градиент — вектор направления наибольшего возрастания; перпендикулярен линии уровня; модуль = скорость роста
  • Антиградиент → движение к минимуму
  • Постоянный шаг: фиксированная величина (learning rate); слишком большой шаг → расходимость/колебания
  • Переменный шаг: длина шага меняется (дробление)
  • Наискорейший: вдоль антиградиента ищет самую глубокую точку (одномерная минимизация)
  • Покоординатный: по очереди вдоль осей координат
  • Функция с оврагом → траектория зигзагом поперёк оврага, медленно вдоль дна
  • Если значение функции перестало убывать → вернуться и уменьшить шаг (h/2)

🎯 Экстремумы и выпуклость

  • Локальный минимум — наименьшее в окрестности; глобальный — на всей области
  • Выпуклая функция — график не выше хорды (f″ ≥ 0); пример: x²; у выпуклой локальный минимум = глобальный
  • Выпуклое множество — отрезок между любыми двумя точками лежит внутри
  • Унимодальная — один глобальный экстремум
  • Производные: (x²)′ = 2x, (kx)′ = k, (const)′ = 0 → f(x)=x²+3x−5 ⇒ f′=2x+3
  • Значение в точке — просто подставить: f(x)=x²−2x, f(3) = 9−6 = 3

📈 Линейное программирование

  • ЛП: целевая функция и все ограничения — линейные
  • Целевая функция — из коэффициентов цели (прибыль/охват на единицу); нормы расхода → в ограничения
  • ⚠️ Следи за единицами: 2000 чел. на 1000 показов = 2 на показ → Z = 5000x₁ + 2x₂
  • Допустимое множество — выпуклый многоугольник; оптимум — в вершине
  • Максимум: линию уровня двигаем по градиенту
  • Симплекс требует канонической формы: ограничения-равенства, переменные ≥ 0
