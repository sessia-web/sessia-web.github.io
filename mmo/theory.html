Постановка задачи оптимизации и основные термины

Оптимизация — выбор наилучшего решения из множества возможных альтернатив без их полного перебора и сравнения; также это процесс, в котором максимизируется количественная характеристика желательного свойства или минимизируется количественная характеристика нежелательного свойства (формулировка лекции). Теория принятия решений — комплексная дисциплина, направленная на разработку методов и средств, помогающих одному или нескольким лицам сделать обоснованный выбор наилучшего из имеющихся вариантов (формулировка лекции).

Математическая постановка задачи

Требуется минимизировать (максимизировать) целевую функцию с учётом ограничений на управляемые переменные (формулировка лекции):

f(x) → min (max), x ∈ X, где:

  • f(x) — целевая функция: функция, которую нужно минимизировать или максимизировать (в задачах ЛП обычно обозначают Z = c1x1 + c2x2 + … → max/min);
  • x = (x1, …, xn) — управляемые переменные: величины, значениями которых мы можем распоряжаться (объёмы выпуска, перевозки и т.п.);
  • X — допустимое множество, заданное ограничениями на управляемые переменные. Ограничения — условия (уравнения и неравенства), которым должно удовлетворять решение;
  • допустимое решение (план) — любой набор переменных, удовлетворяющий всем ограничениям; оптимальное решение — допустимое решение, на котором целевая функция достигает min/max.

Типовая ловушка теста: целевая функция строится из коэффициентов цели (прибыль/охват на единицу), а не из норм расхода ресурсов. Пример: прибыль 20 руб. за P1 и 30 руб. за P2 → Z = 20x1 + 30x2 → max; трудоёмкость и фонд времени идут в ограничение (2x1 + 6x2 ≤ 120), а не в Z.

Задача принятия решений и этапы

Формально задача принятия решений включает: F — формулировку задачи, A — совокупность альтернатив (вариантов), X — совокупность признаков (атрибутов), G — совокупность условий-ограничений, P — предпочтения ЛПР (лица, принимающего решение).

5 этапов процесса принятия решения (формулировка лекции, лекция 2):

  1. появление проблемной ситуации (сформулировать проблему);
  2. необходимость принятия решения;
  3. постановка задачи;
  4. поиск альтернатив метода решения;
  5. выбор метода решения.

Критерий принятия решения — признак, основание, правило принятия решения по оценке чего-либо на соответствие предъявляемым требованиям (формулировка лекции). Требования к совокупности критериев: полнота, разложимость, отсутствие избыточности, прозрачность. Оценка по нескольким критериям — векторная (многокритериальная) оптимизация: учитывается несколько целевых функций.

Ситуации выбора по последствиям: последствия точно известны — условия определённости; имеют вероятностный характер — условия риска; исход неопределён — условия неопределённости.

Классификация задач математического программирования

Классификация по исходным данным, переменным и виду зависимостей (таблица лекции):

  • данные детерминированные, переменные непрерывные, зависимости линейные → ЛП (линейное программирование);
  • данные детерминированные, переменные целочисленные, зависимости линейные → ЦЧП (целочисленное программирование);
  • данные детерминированные, переменные любые, зависимости нелинейные → НЛП (нелинейное программирование);
  • данные случайные → СП (стохастическое программирование).

Определения (формулировки лекции):

  • ЛП — задача оптимизации, в которой целевая функция и все ограничения являются линейными функциями относительно управляемых переменных. Оптимум ЛП достигается в вершине выпуклого многогранника допустимых решений.
  • ЦЧП — раздел задач математического программирования, в котором на все или несколько величин наложено условие целочисленности.
  • НЛП — задача оптимизации, в которой целевая функция или хотя бы одно из ограничений являются нелинейной функцией; в решении могут существовать локальные экстремумы, а допустимое множество может быть невыпуклым.
  • СП — класс задач оптимизации, в котором некоторые параметры модели являются случайными величинами с известными или оцениваемыми распределениями вероятностей.

Типовые задачи по классам — учить наизусть!

На экзамене 3–4 вопроса вида «к какому классу относится задача…»:

  • ЛП: транспортная задача (классический пример ЛП!), задача о смесях (о диете), задача оптимального распределения ресурсов, задача сетевого планирования;
  • ЦЧП: задача о раскрое, задача коммивояжёра, задача о назначениях, задача о рюкзаке;
  • НЛП: задачи наименьших квадратов, задачи оптимального управления, задачи с квадратичными функциями (например, оптимизация портфеля).

Методы решения по классам (для ориентировки): ЛП — симплекс-метод; ЦЧП — метод ветвей и границ, отсечения Гомори; НЛП — градиентные методы, метод Ньютона; динамическое программирование — задачи, где процесс рассматривается последовательно во времени/пространстве (замена оборудования, кратчайший путь, управление запасами).

Структурный и параметрический синтез

Процесс проектирования включает два вида синтеза (формулировки лекции):

  • Структурный синтез — определение перечня типов компонентов и способа их связи между собой, поиск наилучшей структуры и соответствующей ей математической модели. Отвечает на вопрос «из чего и как соединено». Пути решения задач структурной оптимизации: полный перебор (неприемлем при больших размерностях), сокращённый перебор (случайный поиск), экспертная оценка.
  • Параметрический синтез — определение числовых значений параметров (допусков на параметры) элементов в рамках заданной структуры и условий работоспособности. Отвечает на вопрос «какие числа подставить» при уже выбранной структуре. Задачи: определение оптимальных значений параметров, назначение допусков, параметрическая идентификация.
Шпаргалка:
  • Постановка: f(x) → min (max), x ∈ X; f — целевая функция, X — допустимое множество, заданное ограничениями на управляемые переменные.
  • В целевую функцию идут коэффициенты цели (прибыль/цена за единицу), нормы расхода ресурсов — в ограничения: Z = 20x1 + 30x2 → max.
  • 5 этапов принятия решения: проблемная ситуация → необходимость решения → постановка задачи → поиск альтернатив метода → выбор метода.
  • ЛП = транспортная, о смесях, распределение ресурсов, сетевое планирование; ЦЧП = раскрой, коммивояжёр, назначения, рюкзак; НЛП = наименьшие квадраты, оптимальное управление, квадратичные.
  • Случайные исходные данные → стохастическое программирование; целочисленные переменные → ЦЧП; нелинейная f или ограничение → НЛП.
  • Структурный синтез — выбор состава и связей элементов; параметрический — числовые значения параметров при заданной структуре.

Линейное программирование и графический метод

Линейное программирование (ЛП) — задача оптимизации, в которой целевая функция и все ограничения являются линейными функциями относительно управляющих переменных (формулировка лекции). Если хотя бы что-то нелинейно — это уже нелинейное программирование (НЛП); если на переменные наложено условие целочисленности — целочисленное (ЦЧП).

Общая постановка задачи оптимизации: f(x) → min (max), x ∈ X, где f(x) — целевая функция, X — допустимое множество, заданное ограничениями на управляемые переменные. В тесте: функция, которую надо минимизировать/максимизировать, — это именно целевая функция; область, задаваемая всеми ограничениями, — допустимое множество решений; ограничения — условия, которым должно удовлетворять решение.

Запомнить: классический пример задачи ЛП — транспортная задача (также: задача о смесях, оптимальное распределение ресурсов, сетевое планирование). ЛП относится к классу P — решается за полиномиальное время. Основная идея ЛП: поиск оптимума в вершине выпуклого многогранника.

Составление целевой функции из условия — самый частый тип вопроса. Схема одна:

  1. Переменные x1, x2 — количества продукции каждого вида.
  2. Коэффициенты при них — прибыль (эффект) на единицу соответствующего продукта.
  3. Z = c1x1 + c2x2 → max.

Пример из теста: прибыль от P1 — 20 руб., от P2 — 30 руб. ⇒ Z = 20x1 + 30x2 → max. Ловушки в вариантах:

  • Перестановка коэффициентов (Z = 30x1 + 20x2) — проверь, какой коэффициент к какой переменной относится.
  • Подмена прибыли трудоёмкостью: «стул 2 чел.-ч, прибыль 500 руб.; стол 6 чел.-ч, прибыль 1500 руб.» — в целевую функцию идёт прибыль: Z = 500x1 + 1500x2 → max. Трудоёмкость (2 и 6) идёт в ограничение: 2x1 + 6x2 ≤ 120.
  • Ловушка с единицами: «охват на минуту ТВ — 5000 чел., на 1000 показов — 2000 чел.», x2 — показы (в тысячах) ⇒ Z = 5000x1 + 2000x2 → max. Коэффициент берётся в тех же единицах, в которых измеряется переменная (не 2 чел. на один показ, если x2 считается тысячами). Вариант вида (c1+c2)(x1+x2) — всегда неверен.

Ограничения в ЛП — линейные неравенства/равенства: расход ресурса на единицу × количество ≤ запас ресурса, плюс условия неотрицательности x1 ≥ 0, x2 ≥ 0. Каждое неравенство задаёт полуплоскость; их пересечение — допустимый многоугольник (выпуклая область). Выпуклое множество — множество, в котором все точки отрезка, образованного любыми двумя точками множества, также принадлежат этому множеству (формулировка лекции).

Графический метод (для двух переменных):

  1. Построить прямые-границы ограничений, найти допустимый многоугольник.
  2. Провести линию уровня целевой функции: c1x1 + c2x2 = const — на ней Z принимает одно и то же значение.
  3. Построить градиент — вектор (c1; c2) из коэффициентов целевой функции. Градиент функции — вектор, указывающий направление наибольшего возрастания функции в заданной точке и характеризующий скорость её роста (формулировка лекции). Градиент всегда перпендикулярен линии уровня.
  4. Двигать линию уровня параллельно самой себе в направлении градиента (на max) до последней точки касания допустимой области. На минимум — в направлении антиградиента.
  5. Последняя точка касания — оптимум; координаты вершины находят из системы двух уравнений пересекающихся границ, затем подставляют в Z.

Где лежит оптимум. Если оптимальное решение задачи ЛП единственно, оно достигается в вершине допустимого многогранника — не внутри области и не в её «центре». Если линия уровня параллельна одной из сторон многоугольника, оптимум достигается на всём отрезке этой стороны (решений бесконечно много, но среди них по-прежнему есть вершины). Если область не ограничена в направлении градиента — целевая функция не ограничена, оптимума нет.

Шпаргалка:
  • ЛП = и целевая функция, и все ограничения линейны; классический пример — транспортная задача; ЛП ∈ P.
  • Целевая функция максимизации прибыли: Z = (прибыль на единицу) × (количество): Z = c1x1 + c2x2 → max; трудоёмкость/расход ресурсов — в ограничения, не в Z.
  • Следи за единицами: коэффициент должен соответствовать единице измерения переменной (2000 чел. на 1000 показов ⇒ 2000x2, если x2 в тысячах показов).
  • На максимум линию уровня двигают в направлении градиента, на минимум — антиградиента; градиент ⊥ линии уровня.
  • Единственный оптимум ЛП — всегда в вершине допустимого многогранника.
  • Область всех ограничений = допустимое множество решений; функция, которую оптимизируют, = целевая функция.

Симплекс-метод

Примечание: в лекциях курса симплекс-метод подробно не излагался — он упомянут лишь в теме о сложности алгоритмов. Раздел написан по стандартной теории, но критерии оптимальности даны в конвенции, по которой составлены вопросы теста.

Симплекс-метод — универсальный алгоритм решения задач линейного программирования (ЛП): целевая функция и все ограничения линейны. Из лекции: «хотя симплекс-метод является универсальным для решения задач линейного программирования, он позволяет достичь глобального оптимума за разумное время» (формулировка лекции).

Геометрическая идея. Допустимое множество задачи ЛП — выпуклый многогранник. Если оптимальное решение единственно, оно достигается в вершине допустимого многогранника (не внутри и не в центре!). Симплекс-метод — упорядоченный перебор вершин: на каждом шаге переходим к соседней вершине, где целевая функция не хуже, пока улучшение возможно. Это оптимизация «без полного перебора и сравнения» всех вариантов.

Каноническая форма — обязательное требование стандартного симплекс-метода:

  • все ограничения — равенства (не неравенства);
  • все переменные неотрицательны (xj ≥ 0);
  • правые части ограничений неотрицательны (bi ≥ 0).

Неравенство «≤» переводят в равенство добавлением дополнительной (балансовой) переменной: a1x1 + a2x2 ≤ b → a1x1 + a2x2 + s = b, s ≥ 0. Для «≥» переменную вычитают. Дополнительные переменные входят в целевую функцию с коэффициентом 0.

Базис и опорный план. Переменные делятся на базисные (их столько, сколько ограничений; выражены через остальные) и свободные (равны нулю). Опорный план — допустимое решение, отвечающее вершине многогранника. Каждая симплекс-таблица описывает один опорный план.

Оценки (индексная строка). Для каждого столбца j считается оценка:

Δj = cj − zj, где zj = Σ cбаз·aij (сумма произведений коэффициентов целевой функции при базисных переменных на элементы столбца j).

  • У базисных переменных оценки всегда равны нулю (Δj = 0) — частый вопрос теста.
  • Оценка показывает, на сколько изменится Z при вводе единицы переменной xj в план.

Критерий оптимальности (конвенция курса — учить именно так!):

  • Задача на максимум: план оптимален, когда все Δj ≤ 0 (все оценки неположительны). Если есть положительная оценка — план не оптимален, его можно улучшить.
  • Задача на минимум: план оптимален, когда все Δj ≥ 0 (все оценки неотрицательны).

Внимание: в части учебников оценка определяется наоборот, Δj = zj − cj, и знаки в критериях меняются на противоположные. На тесте держитесь конвенции курса: Δj = cj − zj.

Шаг (итерация) симплекс-метода для задачи на максимум:

  1. Проверить оценки. Все Δj ≤ 0 → стоп, план оптимален.
  2. Ведущий (разрешающий) столбец — столбец с наибольшей положительной оценкой: эта переменная вводится в базис.
  3. Ведущая строка — по минимальному симплексному отношению bi/aij среди строк с aij > 0; соответствующая базисная переменная выводится из базиса. Это правило сохраняет неотрицательность плана.
  4. Пересчитать таблицу относительно разрешающего элемента (метод Жордана–Гаусса) и вернуться к п. 1.

Каждая итерация — переход к соседней вершине, значение Z не ухудшается.

Особые случаи:

  • если в ведущем столбце нет положительных элементов (aij ≤ 0 для всех i) — целевая функция не ограничена, оптимума нет;
  • нулевая оценка у небазисной переменной в оптимальной таблице — альтернативный оптимум (решений бесконечно много, ребро многогранника);
  • нулевая базисная переменная — вырожденный план (возможно зацикливание).

Сложность. В худшем случае симплекс-метод имеет экспоненциальную сложность (классический пример — куб Кли–Минти: метод обходит все 2n вершин). Тем не менее на практике он работает быстро. Задача ЛП при этом относится к классу P: «хотя симплекс-метод работает долго в худшем случае, метод эллипсоидов и метод внутренней точки гарантируют полиномиальное время» (формулировка лекции). Полиномиальные альтернативы: метод эллипсоидов (Хачиян, 1979) и метод внутренней точки (Кармаркар, 1984). Но если добавить требование целочисленности переменных, задача становится ЦЛП — NP-трудной.

Шпаргалка:
  • Стандартный симплекс-метод требует каноническую форму: ограничения-равенства + все переменные неотрицательны.
  • Оценки базисных переменных всегда равны нулю.
  • Максимум: оптимум при всех Δj ≤ 0; есть положительная оценка → план не оптимален, можно улучшить; в базис вводят столбец с наибольшей положительной оценкой.
  • Минимум: оптимум, когда все оценки неотрицательны (Δj ≥ 0).
  • Ведущая строка — по минимальному отношению bi/aij (aij > 0).
  • Сложность в худшем случае — экспоненциальная; полиномиальные альтернативы — метод эллипсоидов и метод внутренней точки, поэтому ЛП ∈ P. Оптимум ЛП — в вершине допустимого многогранника.

Транспортная задача: постановка и опорный план

Транспортная задача — это задача об оптимальной перевозке однородного груза от поставщиков к потребителям. Общая постановка транспортной задачи состоит в определении оптимального плана перевозок некоторого однородного груза из m пунктов отправления A1,…,Am в n пунктов назначения B1,…,Bn (формулировка лекции). Критерий оптимальности — минимальная стоимость перевозок всего груза (либо минимальное время доставки). Это НЕ задача о кратчайшем пути и НЕ задача о загрузке транспорта — на тесте эти дистракторы рядом.

Обозначения:

  • cij — тарифы (стоимости перевозки единицы груза) из i-го пункта отправления в j-й пункт назначения; задаются матрицей транспортных издержек (матрицей стоимостей);
  • ai — запасы груза в i-м пункте отправления; bj — потребности j-го пункта назначения;
  • xij — количество единиц груза, перевозимого из i-го пункта в j-й (перевозки) — это переменные задачи.

Математическая модель. Найти минимум целевой функции

Z = ΣiΣj cij·xij → min

при ограничениях: Σj xij = ai (все запасы должны быть перевезены), Σi xij = bj (все потребности должны быть удовлетворены) (формулировка лекции), xij ≥ 0. В развёрнутом виде для 2×2: Z = c11x11 + c12x12 + c21x21 + c22x22 → min — индексы у c и x совпадают попарно (в тесте ловят на перепутанных индексах c12x21).

Число переменных = m × n (каждый поставщик с каждым потребителем). Пример: 3 поставщика и 4 потребителя → 3×4 = 12 переменных xij. Не путать с числом базисных переменных m+n−1!

Всякое неотрицательное решение системы ограничений, определяемое матрицей X=(xij) размера m×n, называют допустимым решением (или планом); план, при котором Z принимает минимальное значение, — оптимальным (формулировка лекции).

Закрытая и открытая модель. Балансовое условие.

  • Для разрешимости транспортной задачи необходимо и достаточно, чтобы запасы груза в пунктах отправления были равны потребностям в грузе в пунктах назначения (формулировка лекции): Σai = Σbj — балансовое условие.
  • Условие выполняется → модель закрытая; не выполняется (Σai ≠ Σbj) → открытая.
  • Открытую приводят к закрытой введением фиктивного поставщика или фиктивного потребителя с тарифами, равными нулю (формулировка лекции). Если Σbj > Σai (потребности больше) — вводят фиктивного поставщика с запасом Σbj − Σai; если Σai > Σbj (запасы больше) — фиктивного потребителя с потребностью Σai − Σbj.
  • Пример: запасы 5, 10, 15 (Σ=30), потребности 10, 15, 20 (Σ=45). 30 ≠ 45 → задача открытая; потребности превышают запасы на 15 → вводим фиктивного поставщика с запасом 45−30 = 15 и нулевыми тарифами.

Алгоритм решения — три этапа (формулировка лекции):

  1. Определение начального допустимого базисного решения (первого опорного плана) — первоначальное распределение поставок.
  2. Проверка полученного решения на оптимальность (методом потенциалов).
  3. Построение последовательных итераций — переход к новому опорному плану, значение целевой функции которого не превышает (меньше или равно) значение предыдущего.

Методы построения опорного плана:

  • Метод северо-западного угла — последовательное заполнение таблицы перевозок, начиная с верхней левой клетки, без учёта стоимости перевозок (формулировка лекции). В северо-западную (левую верхнюю) клетку записываем наименьшее из чисел — запаса строки и потребности столбца (min(ai, bj)); исчерпанную строку/столбец вычёркиваем (прочерки), пересчитываем остатки и переходим к следующей северо-западной клетке. Тарифы игнорируются, поэтому план обычно далёк от оптимума.
  • Метод минимальной цены (наименьшей стоимости) — из таблицы выбирают клетку с наименьшим тарифом и помещают в неё наименьшее из ai и bj; далее сокращают строку или столбец, ресурс которого израсходован, и выбирают следующую клетку с наименьшей стоимостью (формулировка лекции). Учитывает тарифы, поэтому даёт план ближе к оптимальному (в примере лекции: СЗ-угол дал F=3630, наименьшая стоимость F=3410 при оптимуме 3340).

Невырожденность. В невырожденном опорном плане число занятых (заполненных) клеток равно m + n − 1, где m — число поставщиков (складов), n — число потребителей (магазинов). Это же число — количество базисных переменных. Вырожденность — ситуация, когда занятых клеток оказалось меньше m + n − 1 (тогда в одну из свободных клеток ставят «нулевую поставку»).

Шпаргалка:
  • Z = Σcij·xij → min; переменных всего m × n (3×4 = 12), базисных — m + n − 1.
  • Балансовое условие Σai = Σbj — необходимое и достаточное условие разрешимости; выполняется → закрытая, нет → открытая.
  • Открытая → закрытая: фиктивный поставщик/потребитель, тарифы нулевые, объём = |Σai − Σbj| (запасы 5-10-15, потребности 10-15-20 → фиктивный поставщик на 15).
  • СЗ-угол: с левого верхнего угла, min(запас, потребность), без учёта тарифов; наименьшая стоимость: клетка с min тарифом — план ближе к оптимуму.
  • Невырожденный опорный план: занятых клеток ровно m + n − 1; меньше — вырожденность.
  • Три этапа: опорный план → проверка на оптимальность (потенциалы) → итерации с невозрастающей Z.

Метод потенциалов

Метод потенциалов — метод проверки оптимальности опорного плана транспортной задачи и его последовательного улучшения. Не путать с методами построения начального плана (северо-западный угол, минимальная цена): те дают первый опорный план, а потенциалы отвечают на вопрос «оптимален ли он и как его улучшить». В тесте это спрашивают прямо: «какой метод служит для проверки оптимальности плана транспортной задачи?» — метод потенциалов.

Потенциалы — это числа ui и vj, сопоставляемые каждому поставщику Ai и каждому потребителю Bj для проверки оптимальности плана (формулировка лекции). Это не объёмы грузов, не тарифы и не запасы — именно вспомогательные числа.

Критерий оптимальности (теорема). Чтобы допустимый план перевозок был оптимальным, необходимо и достаточно, чтобы существовала система из m+n чисел ui, vj (потенциальная система), удовлетворяющих условиям:

  • ui + vj = cij — для занятых (базисных) клеток (xij > 0);
  • ui + vj ≤ cij — для свободных клеток (xij = 0).

Алгоритм метода потенциалов:

  1. Найден опорный план (северо-западным углом или минимальной ценой). В невырожденном плане занятых клеток ровно m + n − 1.
  2. Для базисных клеток составляем систему ui + vj = cij. Уравнений m + n − 1, неизвестных m + n, поэтому один потенциал задаём произвольно (обычно u1 = 0), остальные вычисляются из системы (формулировка лекции).
  3. Для каждой свободной клетки вычисляем оценку: ΔCij = cij − (ui + vj).
  4. Если все ΔCij ≥ 0 (неотрицательны) — план оптимален. Если хотя бы одна ΔCij < 0 — план не оптимален и улучшается переносом по циклу, соответствующему этой свободной клетке.

Внимание, знак оценки — ловушка. В конспекте лекций оценка определена как ΔCij = cij − (ui + vj), и признак оптимальности — все оценки неотрицательны (именно этот вариант правильный в тесте). В презентации то же правило записано через wij = ui + vj − cij с условием оптимальности wij ≤ 0. Суть одна — знак зависит от того, как определили оценку. На тесте ориентируйтесь на конвенцию конспекта: признак оптимальности — все оценки (Δj) неотрицательны.

Улучшение плана (перенос по циклу):

  • Среди свободных клеток с отрицательными ΔCij выбираем ту, у которой ΔCij наименьшая (самая отрицательная) (формулировка лекции).
  • Для этой клетки строим цикл (контур) — замкнутую ломаную из горизонтальных и вертикальных отрезков. Одна вершина — в свободной клетке, остальные — в занятых; число вершин всегда чётное (формулировка лекции).
  • Свободной вершине придаём знак «+», далее знаки вершин чередуются (−, +, −, …).
  • По циклу переносим величину, равную минимальной перевозке среди «минусовых» вершин: её прибавляем в клетках «+» и вычитаем в клетках «−». Одна из занятых клеток освобождается, свободная становится занятой.
  • Пересчитываем потенциалы и оценки заново; итерации повторяем, пока все оценки не станут неотрицательными. Целевая функция на каждом шаге не возрастает.

Вырожденность: план вырожден, если число занятых клеток меньше m + n − 1. Тогда в недостающие клетки ставят «нулевые» перевозки (базисный нуль), чтобы система потенциалов решалась.

Связь с двойственностью: потенциалы ui, vj — это двойственные переменные транспортной задачи как задачи ЛП, а критерий оптимальности — её условия двойственности. Аналогия с симплекс-методом: в задаче минимизации план оптимален, когда все оценки неотрицательны; отрицательная оценка показывает, что план можно улучшить.

Шпаргалка:
  • Проверка оптимальности плана транспортной задачи — метод потенциалов (северо-западный угол и минимальная цена — только начальный план).
  • Потенциалы — числа, сопоставляемые поставщикам и потребителям для проверки оптимальности.
  • Для занятых клеток: Ui + Vj = Cij (именно сумма!). Один потенциал задаём произвольно (u1 = 0).
  • Оценка свободной клетки: ΔCij = cij − (ui + vj). Признак оптимальности — все оценки (Δj) неотрицательны.
  • Есть ΔCij < 0 → берём клетку с наименьшей оценкой и переносим по циклу: одна вершина в свободной клетке, остальные в занятых, число вершин чётное, знаки чередуются от «+» в свободной.
  • Занятых клеток в невырожденном плане m + n − 1; меньше — план вырожден.

Задача коммивояжёра и задача о рюкзаке

Обе задачи — классика целочисленного программирования (ЦЧП). В лекциях типовые задачи распределены так: «Целочисленное программирование: задача о раскрое, задача коммивояжера, задача о назначениях, задача о рюкзаке» (формулировка лекции). Не путать: классический пример линейного программирования — транспортная задача, а коммивояжёр и рюкзак в тесте там стоят как ловушки-дистракторы.

Задача коммивояжёра (TSP)

«Коммивояжер должен объехать N городов с целью осуществления продажи своих товаров. Все N городов соединены дорогами по принципу „каждый с каждым". Известна стоимость проезда между двумя любыми городами. Найти оптимальный маршрут движения так, чтобы побывать во всех городах и при этом иметь минимальные затраты на дорогу» (формулировка лекции). Иначе: найти кратчайший замкнутый маршрут, проходящий через каждый город не более одного раза, с возвратом в исходный пункт.

  • Исходные данные: перечень городов и матрица стоимостей — двумерный массив N×N с элементами cij, равными стоимости проезда из города i в город j.
  • Целевая функция (общий вид): минимизировать суммарную стоимость маршрута — min Σ cijxij, где xij = 1, если участок i→j входит в маршрут, и 0 иначе. Переменные булевы ⇒ это задача ЦЧП.
  • Число маршрутов: различных замкнутых маршрутов для n городов — (n−1)!/2. Запомнить: не n!, не n2, не 2n.
  • Класс сложности: задача NP-трудная (в тесте именно этот вариант; не P, не co-NP, не «полиномиальная»). Обоснование лекции: количество маршрутов (n−1)!/2, алгоритмов быстрее экспоненциальных не найдено. В классификации классов лекция приводит коммивояжёра и как пример NPC (NP-полной) задачи — самой трудной внутри NP.

Методы решения:

  1. Точное решение — полный перебор всех возможных маршрутов («лобовой» способ: генерируем все перестановки городов, отбрасываем недопустимые, среди оставшихся берём кратчайший). Сложность O(n!) — факториальная, наибольшая среди известных типов.
  2. Метод ветвей и границ — точное решение задачи небольшой размерности (типовой ответ теста).
  3. Приближённые (эвристики): жадный алгоритм «ближайшего соседа» — берём каждый раз ближайший город. Полиномиальная сложность, но результат не обязательно оптимален ⇒ точным решением в тесте НЕ является (как и градиентный спуск).

Задача о рюкзаке

Формулировка: «наполнить рюкзак самыми ценными вещами, но ограниченными по весу» (формулировка лекции). Есть n предметов, у каждого ценность ci и вес wi; вместимость рюкзака W ограничена.

  • Переменные 0/1: xi = 1, если предмет i кладём в рюкзак, xi = 0 — если нет (булевы переменные «да/нет»).
  • Целевая функция (общий вид): максимизировать суммарную ценность — max Σ cixi.
  • Ограничение: суммарный вес не превышает вместимость — Σ wixi ≤ W, xi ∈ {0; 1}.
  • Связь с ЦЧП: целевая функция и ограничение линейны, но переменные целочисленные (0 или 1) — это задача целочисленного линейного программирования. Именно условие целочисленности делает задачу комбинаторной (нужно перебирать варианты) и NP-трудной.
  • Место в классах сложности: рюкзак — пример задачи класса NP: «проверить, что набор предметов укладывается в вес, легко» (формулировка лекции), а вот найти решение может быть долго.

Методы решения ЦЧП в целом: метод ветвей и границ, метод отсечений (Гомори), полный перебор для малых размерностей.

Шпаргалка:
  • Число замкнутых маршрутов коммивояжёра для n городов — (n−1)!/2; полный перебор имеет сложность O(n!).
  • Задача коммивояжёра — NP-трудная (вариант «P» / «co-NP» / «полиномиальная» — неверно).
  • Точное решение TSP: полный перебор всех возможных маршрутов; для небольшой размерности — метод ветвей и границ. Жадный алгоритм и «ближайший сосед» — только приближённые.
  • Рюкзак: переменные xi ∈ {0;1}, max Σcixi при Σwixi ≤ W — задача целочисленного (0/1) программирования.
  • Коммивояжёр, рюкзак, раскрой, назначения — примеры ЦЧП; классический пример ЛП — транспортная задача.
  • ЦЧП NP-трудно именно из-за условия целочисленности: оно делает задачу комбинаторной.

Сложность алгоритмов и классы P/NP

Теория сложности классифицирует уровни сложности алгоритмов решения задач, а также сложность самих задач (формулировка лекции). Рассматривается минимальное время и объём памяти для решения самого трудного варианта проблемы на теоретическом компьютере — машине Тьюринга.

Сложность алгоритма — комплексное свойство, позволяющее определить затраты на его реализацию; определяется вычислительными мощностями, требуемыми для выполнения (формулировка лекции). В минимальном варианте описывается двумя параметрами:

  • T — временная сложность (время работы);
  • S — пространственная сложность (объём памяти).

Оба параметра зависят от размера входных данных n (размера массива данных). Сложность представляется функцией f(n), которая даёт верхнюю границу максимального числа основных операций (сложения, сравнения и т.д.), которые должен выполнить алгоритм.

Асимптотические обозначения (отбрасываем константы и младшие члены, оставляя главную тенденцию):

  • O — верхняя граница в худшем случае (формулировка лекции);
  • Ω — нижняя граница в лучшем случае;
  • Θ — точная граница, когда верхняя и нижняя совпадают.

Виды временной сложности по возрастанию (какая больше/меньше — вопрос экзамена!):

  1. Постоянный — O(1): «один к одному», не зависит от n;
  2. Логарифмический — O(log n): большая задача сводится к набору меньших (размер уменьшается на каждом шаге), пример — бинарный поиск;
  3. Линейный — O(n): обрабатывается каждый входной элемент;
  4. Квадратный — O(n2): обработка пар элементов;
  5. Кубический, полиномиальный — O(n3), O(nk): обработка троек элементов и т.д.;
  6. Экспоненциальный — O(cf(n)), где c — константа, f(n) — полиномиальная функция; перебор всех возможных решений;
  7. Факториальный — O(n!): перебор всех сочетаний; обладает наибольшей временной сложностью среди всех известных типов (формулировка лекции).

Итого: O(1) < O(log n) < O(n) < O(n2) < O(n3) < O(cn) < O(n!).

Класс P — класс задач с полиномиальной сложностью (полиномиально-решаемых). Задачи свободно решаются на ЭВМ, время решения примерно пропорционально объёму исходных данных (формулировка лекции). Задача полиномиальна, если существует константа k и алгоритм, решающий её за время O(nk), где n — длина входа в битах.

Свойства класса P:

  • для большинства реальных задач k ≤ 6;
  • инвариантность в рамках широкого класса моделей вычислений;
  • естественная замкнутость: сумма или произведение полиномов — тоже полином.

Примеры задач класса P: линейное программирование (симплекс-метод в худшем случае экспоненциален, но метод эллипсоидов и метод внутренней точки — полиномиальны), выпуклая оптимизация и градиентный метод (сходится к оптимуму экспоненциально быстро), задача кратчайшего пути (алгоритм Дейкстры), транспортная задача (сводится к ЛП), сортировка массива.

Класс NP — класс задач с полиномиально проверяемым решением: найти ответ может быть очень трудно, но если решение уже есть, его правильность проверяется за полиномиальное время. Алгоритм, проверяющий решение, относится к классу P. Пример: задача о рюкзаке — проверить, что набор предметов укладывается в вес, легко.

Любая задача из P принадлежит и NP (проверка может состоять в повторном решении). Основная проблема теории сложности — совпадают ли P и NP; ни совпадение, ни несовпадение не доказаны, большинство исследователей предполагают P ≠ NP.

Класс co-NP — задачи, у которых за полиномиальное время проверяется ответ «НЕТ» (задача принадлежит co-NP, если её дополнение принадлежит NP).

NP-трудная задача — задача H, к которой любую задачу из NP можно свести за полиномиальное время. При этом H может быть какой угодно сложности (даже неразрешимой) и не обязана принадлежать классу NP.

NP-полная задача (NPC) — требует выполнения двух условий (формулировка лекции):

  1. задача принадлежит классу NP;
  2. к ней полиномиально сводятся все задачи из NP (т.е. она NP-трудная).

Кратко: NP-полная = NP-трудная И принадлежит NP. NPC — «самые сложные» задачи внутри NP. Теорема: если хотя бы одна NPC-задача решается за полиномиальное время, то P = NP (все задачи NP станут «быстрыми»). Доказаны сотни NP-полных задач, но ни для одной не найден полиномиальный алгоритм.

Примеры NP-трудных задач:

  • задача коммивояжера (TSP): объехать N городов, соединённых «каждый с каждым», с минимальными затратами; лобовой перебор — факториальный алгоритм O(n!), число замкнутых маршрутов равно (n−1)!/2; точное решение — полный перебор (для малых размерностей — метод ветвей и границ), приближённое (жадный алгоритм ближайшего соседа) имеет полиномиальную сложность;
  • целочисленное линейное программирование, в т.ч. задача о рюкзаке — условие целочисленности делает задачу комбинаторной;
  • задача о вершинном покрытии / клике (камеры наблюдения, покрывающие все переходы).

Нерешаемые (трудные) — проблемы, которые невозможно решить за полиномиальное время (формулировка лекции). Неразрешимые — сильнее: Алан Тьюринг доказал, что некоторые проблемы принципиально неразрешимы, то есть невозможно создать алгоритм их решения вообще, независимо от временной сложности.

Шпаргалка:
  • Порядок сложностей: O(1) < O(log n) < O(n) < O(n2) < O(n3) < экспоненциальная < факториальная O(n!) — наибольшая.
  • O — верхняя граница в худшем случае, Ω — нижняя в лучшем, Θ — точная (совпадают). T — время, S — память.
  • P = быстро решаем O(nk); NP = быстро проверяем готовое решение; NP-полная = NP-трудная И принадлежит NP; co-NP = быстрая проверка ответа «НЕТ».
  • Коммивояжер — NP-трудная, перебор O(n!), маршрутов (n−1)!/2; точно решается полным перебором / методом ветвей и границ.
  • Класс P: ЛП, транспортная задача, кратчайший путь, выпуклая/градиентная оптимизация. NP-трудные: коммивояжер, целочисленное ЛП, рюкзак.
  • Симплекс-метод в худшем случае — экспоненциальный; Тьюринг: есть принципиально неразрешимые задачи (алгоритма не существует).

Теория принятия решений

Теория принятия решений (ТПР) — комплексная дисциплина, направленная на разработку методов и средств, помогающих одному или нескольким лицам делать обоснованный выбор наилучшего из имеющихся вариантов (формулировка лекции). В тесте правильный вариант: «дисциплина, разрабатывающая методы обоснованного выбора наилучшего варианта» (не «наука о программировании», не «раздел статистики»).

Принятие решения (выбор) — действие над множеством альтернатив, в результате которого получается подмножество выбранных альтернатив (обычно одна). Выбор тесно связан с оптимизацией: оптимизация — это выбор оптимальной альтернативы. Выбор возникает только тогда, когда есть не менее двух возможных вариантов.

Проблема принятия решений — возникает в тех случаях, когда проблема (задача) настолько усложняется, что для её формулирования (постановки) не может быть сразу применён подходящий аппарат формализации, когда процесс постановки задачи требует участия специалистов различных областей (формулировка лекции). Сама постановка задачи становится проблемой. Для принятия решения нужно получить выражение, связывающее цель со средствами её достижения с помощью критериев оценки достижимости цели (это выражение называют критерием эффективности, целевой функцией, функцией цели).

Проблемная ситуация — несоответствие между существующим и требуемым (целевым) состоянием системы при данном состоянии среды — именно этот вариант правильный в тесте. Её признаки: высокая неопределённость (недостаток информации), противоречивость, слабая структурированность, слабая формализуемость.

5 этапов процесса принятия решения (формулировка лекции — учить в этом порядке):

  1. появление проблемной ситуации (сформулировать проблему);
  2. необходимость принятия решения;
  3. постановка задачи;
  4. поиск альтернатив метода решения;
  5. выбор метода решения.

Процедура многоэтапная и итеративная: если лучший вариант не найден, возвращаются на предыдущие этапы (меняют формулировку, собирают информацию, расширяют/сужают множество альтернатив). Именно на стадии постановки задачи определяют совокупность возможных вариантов решения (альтернатив).

Формальная запись задачи принятия решения: D = <F, A, X, G, P> — вопрос теста, знать каждую букву:

  • F — формулировка задачи: содержательное описание проблемы (при необходимости — модельное представление), определение цели и требования к виду окончательного результата;
  • A — совокупность возможных вариантов (альтернатив), из которых производится выбор (объекты, действия, способы достижения цели); теоретически может быть бесконечной;
  • X — совокупность признаков (атрибутов, параметров), описывающих варианты и их отличительные особенности: объективные (можно измерить/посчитать) и субъективные (оценки по критериям);
  • G — совокупность условий, ограничивающих область допустимых вариантов; задаются содержательно или как формальные требования к вариантам и их признакам;
  • P — предпочтения ЛПР (лица/лиц, принимающих решение) — основа для оценки и сравнения вариантов; часто часть предпочтений превращают в ограничения.

Факторы проблемной ситуации: управляемые (зависят от ЛПР: цели, альтернативы, субъективные оценки) и неуправляемые (объективные признаки, часть ограничений); по характеру — детерминированные, вероятностные (стохастические), неопределённые.

Критерий — признак, основание, правило принятия решения по оценке чего-либо на соответствие предъявляемым требованиям (формулировка лекции). В тесте: «признак, на основании которого производится оценка вариантов» (не ограничение, не метод, не допустимое множество). Критерии — «количественные модели качественных целей»: цели просто называются, а критерии выражаются в шкалах измерения.

4 требования к совокупности критериев (вопрос теста — среди дистракторов «субъективность», «противоречивость», «неопределённость»; правильное — полнота):

  • полнота — набор критериев отражает все существенные аспекты проблемы и качества её решения;
  • разложимость — состав критериев должен упрощать описание и анализ проблемы;
  • отсутствие избыточности (неизбыточность) — число критериев минимально, критерии не дублируют друг друга;
  • прозрачность — содержание и смысл критериев, формулировки градаций оценок однозначно понимаются всеми участниками.

Ситуации (условия) выбора — по характеру последствий:

  • определённость — последствия выбора точно известны;
  • риск — последствия имеют вероятностный характер (вопрос теста: «вероятностный характер» = условия риска, не путать с неопределённостью!);
  • неопределённость — исход неопределён (вероятности неизвестны).

Также различают: оценку по одному или нескольким критериям и режим выбора — однократный (разовый) или повторяющийся.

Многокритериальная (векторная) оптимизация — оптимизация, при которой оценка альтернатив осуществляется по нескольким критериям, т.е. учитывается несколько целевых функций одновременно (ответ теста). Критерии могут быть количественными (пример — полезность) и качественными. Здесь обычно нет единственного «лучшего по всем критериям» решения — ищут компромиссные варианты.

Шпаргалка:
  • ТПР = дисциплина для обоснованного выбора наилучшего варианта из имеющихся.
  • 5 этапов: проблемная ситуация → необходимость решения → постановка задачи → поиск альтернатив → выбор метода. Альтернативы определяют на стадии постановки.
  • D = <F, A, X, G, P>: F — формулировка, A — альтернативы, X — признаки (атрибуты), G — ограничения, P — предпочтения ЛПР.
  • Критерий = признак для оценки вариантов. 4 требования к совокупности критериев: полнота, разложимость, отсутствие избыточности, прозрачность.
  • Последствия точно известны — определённость; вероятностный характер — риск; неизвестный исход — неопределённость.
  • Многокритериальная (векторная) оптимизация = несколько целевых функций (не много переменных и не много ограничений!).
  • Проблемная ситуация = несоответствие существующего и требуемого состояния системы.

Градиентные методы, экстремумы и выпуклость

Градиент и антиградиент

Градиент функции — вектор, указывающий направление наибольшего возрастания функции в заданной точке и характеризующий скорость этого роста (формулировка лекции). Это многомерное обобщение производной: ∇f = (∂f/∂x1, ∂f/∂x2, …, ∂f/∂xn).

Два важных геометрических свойства градиента (формулировка лекции):

  • Направление: градиент ∇f в точке указывает направление наибольшего возрастания функции в этой точке;
  • Величина: модуль градиента |∇f| равен скорости возрастания функции в направлении градиента.

Для функции двух переменных градиент всегда перпендикулярен линии уровня, проходящей через данную точку: на поверхности z = f(x, y) он показывает направление самого крутого подъёма. Антиградиент (−∇f) — противоположный вектор, направление наискорейшего убывания; именно по нему движутся при поиске минимума.

Пример: f(x, y) = x2 + 3y2 → ∇f = (2x; 6y); в точке (1; 1) градиент равен (2; 6), антиградиент (−2; −6).

Семейство градиентных методов (отличия — вопрос экзамена)

Все методы строят последовательность точек xk+1 = xk − h·∇f(xk), где h — градиентный шаг. Различаются они способом выбора шага и направления:

  1. Градиентный спуск с постоянным шагом. Движение к минимуму всегда происходит по антиградиенту, в сторону наискорейшего локального убывания функции, на одну и ту же заранее заданную величину (формулировка лекции). Шаг — константа (learning rate). Мал — спуск непозволительно долгий, «застревает» на плато; велик — точка перепрыгивает дно и колеблется с нарастающей амплитудой.
  2. Градиентный спуск с переменным шагом. Перемещение по антиградиенту, но длина этого перемещения не фиксирована (формулировка лекции) — шаг подстраивается на каждой итерации: априорное дробление (уменьшение по заданному закону) или условие дробления (правило Армихо: если функция уменьшилась недостаточно, шаг дробится).
  3. Метод наискорейшего спуска. Усовершенствованный метод с переменным шагом: метод не просто ищет точку вдоль выбранной прямой, а ищет сразу самую глубокую (абсолютный оптимум) (формулировка лекции). Шаг находится одномерной минимизацией вдоль антиградиента: решаем dφ(h)/dh = 0, где φ(h) = f(xk − h·∇f(xk)). Каждый следующий шаг перпендикулярен предыдущему.
  4. Покоординатный спуск. Минимизация ведётся по очереди вдоль каждой координатной оси (сначала по x1 при фиксированных остальных, потом по x2 и т.д.). Градиент целиком не вычисляется — метод пригоден, когда производные по всем переменным сразу считать трудно.

Алгоритм наискорейшего спуска (формулировка лекции): 1) находим градиент функции; 2) задаём начальную точку x0; 3) вычисляем ∇f(x0); 4) записываем x1 = x0 − h·∇f(x0); 5) подставляем координаты x1 в функцию, получаем φ(h); 6) решая dφ/dh = 0, находим h, при котором φ(h) имеет min; 7) подставляем h; 8) сравниваем f(x1) и f(x0) — необходимо f(x1) < f(x0); 9) повторяем.

Овражные функции: в узких длинных «оврагах» траектория спуска — зигзаг: точка колеблется от одного склона оврага к другому и продвигается ко дну очень медленно.

Скорость обучения (learning rate) — это величина шага в направлении антиградиента. Слишком большая скорость обучения → алгоритм может расходиться или колебаться вокруг минимума; слишком маленькая → сходимость чрезмерно медленная.

Локальный и глобальный экстремум, унимодальность

  • x* — точка локального максимума, если существует окрестность точки x*, в которой f(x*) ≥ f(x) для всех x из этой окрестности; локального минимума — если f(x*) ≤ f(x) в некоторой окрестности (формулировка лекции).
  • x* — точка глобального максимума (минимума), если f(x*) ≥ f(x) (соответственно f(x*) ≤ f(x)) для любого x из всей области определения.
  • Функция называется унимодальной, если у неё имеется только один глобальный экстремум, в других случаях — многоэкстремальной (формулировка лекции).

Выпуклость

  • Выпуклое множество — множество, в котором все точки отрезка, образуемого любыми двумя точками данного множества, также принадлежат данному множеству (формулировка лекции).
  • Выпуклая функция — функция, график которой лежит не выше хорды, соединяющей любые две его точки (надграфик — выпуклое множество). Признак: у дважды дифференцируемой функции вторая производная неотрицательна: f″(x) ≥ 0.
  • Главное свойство для оптимизации: у выпуклой функции любой локальный минимум является глобальным (у вогнутой — локальный максимум глобален). Любая стационарная точка выпуклой функции — глобальный экстремум.
  • Поэтому выпуклая оптимизация (минимизация выпуклой функции на выпуклом множестве) относится к классу P: градиентные методы сходятся к глобальному минимуму экспоненциально быстро.

Простейшие вычисления — будут на экзамене

Правила: (xn)′ = n·xn−1, (c·f)′ = c·f′, константа′ = 0. Значение функции в точке — простая подстановка.

  • f(x) = x2 − 4x + 1 → f′(x) = 2x − 4; f′(3) = 2; значение f(3) = 9 − 12 + 1 = −2.
  • f(x) = x3 − 2x → f′(x) = 3x2 − 2; f′(2) = 10; f(2) = 8 − 4 = 4.
  • Точка-кандидат на экстремум: f′(x) = 0 (для f(x) = x2 − 4x + 1 это x = 2, минимум, так как f″ = 2 > 0).
Шпаргалка:
  • Градиент = направление наибольшего возрастания, |∇f| = скорость роста, всегда ⊥ линии уровня; минимум ищут по антиградиенту.
  • Постоянный шаг — фиксированная величина; переменный шаг — длина не фиксирована; наискорейший спуск — шаг из одномерной минимизации (самая глубокая точка вдоль прямой); покоординатный — по очереди вдоль осей.
  • Скорость обучения = величина шага в направлении антиградиента; слишком большая → метод расходится или колеблется.
  • На овражной функции траектория спуска — зигзаг между склонами оврага.
  • Выпуклая функция: график не выше хорды; её локальный минимум = глобальный. Унимодальная функция — один глобальный экстремум.
  • (xn)′ = n·xn−1: у f(x) = x2 − 4x + 1 производная 2x − 4, в точке x = 3 равна 2, а f(3) = −2.
